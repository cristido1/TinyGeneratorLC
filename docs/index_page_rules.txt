OBIETTIVO
Definire lo standard unico per pagine Index (lista/griglia) nel progetto.

STANDARD OBBLIGATORIO
- Tutte le pagine lista/griglia devono usare l'API CRUD generica esistente:
  - `/api/crud/{table}/query`
  - `/api/crud/{table}/{id}`
  - endpoint correlati del `BaseCrudController`
- Il frontend delle griglie deve usare Vue + VuePrime (DataTable/controlli correlati).
- Non introdurre nuove griglie DataTables/jQuery per nuove pagine.

PAGINE SEMPLICI
- Se la tabella ha CRUD standard e non richiede logica UI specifica, usare obbligatoriamente:
  - `Pages/Shared/Index`
- La pagina `Shared/Index` deve restare la scelta predefinita per:
  - gestione colonne/filtri/paginazione
  - metadati da interfacce condivise (es. `IActiveFlag`, `IImageFile`, `ISoundFile`, `IVideoFile`)
  - operazioni CRUD base tramite API generica
  - uso di relazioni FK/integrità referenziale per lookup e campi collegati
- `Pages/Shared/Index` NON deve avere cablature hardcoded specifiche di singole tabelle o singoli campi.
- Ogni estensione deve essere implementata in forma generica, guidata da:
  - interfacce delle entità
  - metadati FK/relazioni dal backend generico
  - convenzioni condivise di naming, non eccezioni ad-hoc

QUANDO CREARE UNA INDEX DEDICATA
- Creare una pagina Index dedicata solo se esiste un requisito non coperto da `Shared/Index`, ad esempio:
  - workflow multi-step specifici
  - visualizzazioni aggregate o dashboard custom
  - azioni di dominio non riconducibili a CRUD standard
- In tal caso, anche la pagina dedicata deve comunque usare API generica e componenti VuePrime, salvo eccezioni approvate esplicitamente.

NOTE IMPLEMENTATIVE
- Evitare duplicazione di logica CRUD lato pagina: centralizzare su API generica.
- Mantenere naming campi coerente con interfacce modello per sfruttare automazioni della pagina shared.
- Prima di modificare/creare una Index, verificare se l'obiettivo è raggiungibile estendendo `Shared/Index`.
