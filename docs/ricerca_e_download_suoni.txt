- Tipi ammessi: FX, AMB (ambience). Saltare MUSIC.

====================================================================
3) FLUSSO PRINCIPALE (1 record alla volta)
====================================================================

Metodo entry-point suggerito:
SoundSearchService.ProcessOneMissingSound(int missingId) -> SoundSearchResult

Pseudoflow:

1) Carica record mancante (missing) da DB
2) Se missing.Type == "music" -> return Skipped
3) Normalizza input:
   - prompt = missing.Prompt
   - tags = parse CSV missing.Tags
4) Costruisci “SearchPlan”:
   - Determina se FX o AMB
   - Calcola range durata:
       FX: duration <= MaxDurationSecondsFx
       AMB: duration <= MaxDurationSecondsAmb
   - Genera query in inglese + fallback in italiano:
       q1 = Translate/Map IT->EN (semplice dizionario)
       q2 = keywords dai tag (EN)
       q3 = prompt keywords puliti
   - MaxQueryTerms: troncare
5) Per ogni fonte abilitata (ordine consigliato):
   A) Freesound
   B) Mixkit
   C) Pixabay
   (ordine motivazione: Freesound ha tag/metadati migliori; Mixkit qualità buona; Pixabay più generico)
6) Per ogni fonte:
   - esegui ricerca (max N candidati)
   - valuta e ordina candidati (scoring semplice)
   - seleziona best candidate
   - se best candidate valido:
       - scarica file
       - salva su disco (cartella per tipo + slug)
       - inserisci in tabella sounds
       - aggiorna record mancante (Status="found")
       - return Found
7) Se nessuna fonte trova -> aggiorna record mancante (Status="failed" + motivo) e return NotFound

4) GENERAZIONE QUERY (IT->EN semplice, no AI)
====================================================================

Niente LLM: usare mapper deterministico + tokenizzazione.
Implementare:
- Tokenize: lowercase, rimuovi punteggiatura, stopwords IT base (es: "che", "si", "un", "una", "dal", "della", "sul", "sopra", "loro", ecc.)
- Stemming leggero opzionale (non indispensabile)
- Dizionario sinonimi IT->EN per parole frequenti nel tuo dominio:
  porta->door, schianta->slam, urto->impact, colpo->hit, esplosione->explosion, vibrazione->vibration, motore->engine, ponte->bridge/deck, nave->ship, tunnel->tunnel, grido->scream/shout, tonfo->thud, boato->boom, metallo->metal, pavimento->floor, frena->brake, pneumatici->tires, stridio->screech, banco->bench, scaffale->shelf, crolla->collapse, legno->wood, ecc.
- Per FX preferire termini “one-shot”: hit, slam, thud, impact, crash, boom, rumble.
- Per AMB preferire termini “ambience”: ambience, room tone, background, interior, exterior, engine hum, ventilation, crowd, wind.

Generare fino a 3 query:
- QueryPrimary: da tag tradotti + 1-2 termini dal prompt
- QuerySecondary: da prompt tradotto (top terms)
- QueryItalianFallback: prompt originale pulito

====================================================================
5) SCORING CANDIDATI (semplice)
====================================================================

Score = TagMatch + PromptMatch + DurationFit + FormatPreference + SourceBias

- TagMatch: conteggio match keyword (0..10)
- PromptMatch: 0..5
- DurationFit:
   - FX: preferisci 0.3..5 sec (bonus), penalizza > MaxDurationSecondsFx
   - AMB: preferisci 20..180 sec (bonus), penalizza < MinDurationSeconds
- FormatPreference: +1 se wav/flac, +0.5 se mp3
- SourceBias: Freesound +1, Mixkit +0.5, Pixabay +0 (configurabile)

Se candidate non ha licenza consentita -> scarta.
Se URL già presente in DB (SkipIfAlreadyInSounds) -> scarta.

====================================================================
6) IMPLEMENTAZIONE FREESOUND (API)
====================================================================

6.1 Search
Endpoint:
GET {BaseUrl}/search/text/
Params:
- query = <query>
- fields = id,name,previews,duration,tags,license,username,download,images,url
- filter = duration:[0 TO X] (X dipende da FX/AMB)
- page_size = PageSize
Headers:
- Authorization: Token <ApiKey>

Note:
- Freesound per download spesso richiede endpoint download dedicato e token.
- Usare “previews” se il download diretto è limitato (previews sono mp3/ogg).
- Se vuoi WAV, serve download ufficiale e permessi; altrimenti usare preview mp3 (accettabile per “prima bozza”).

6.2 Candidate mapping
Estrarre:
- ExternalId = id
- Title = name
- SourceUrl = url
- DownloadUrl = download (se presente) altrimenti previews.preview-hq-mp3 o simile
- Tags = tags (array)
- DurationSeconds = duration
- License = license
- Author = username

6.3 Download
- Se usi previews: GET preview url -> salva file .mp3
- Se usi download: chiamare URL download con header Authorization.

9) DOWNLOAD + SALVATAGGIO FILE
====================================================================

9.1 Naming e cartelle
BaseFolder = SoundSearch.DownloadFolder

Path:
- FX:  <Base>\fx\<slug_prompt_or_tags>\<source>_<externalId>.<ext>
- AMB: <Base>\amb\<slug_prompt_or_tags>\<source>_<externalId>.<ext>

Slug:
- da prompt/tags, max 40 char, solo [a-z0-9-_]

9.2 Dedup
- Se possibile calcolare SHA256 del contenuto e salvare in sounds.HashSha256
- Se hash già esiste -> non inserire doppio; aggiornare missing come “found_duplicate” e puntare al suono esistente.

9.3 Validazioni minime
- file size > 5KB
- estensione valida
- durata se disponibile (se non disponibile, non bloccare)

10) INSERIMENTO IN DB (tabella sounds)
====================================================================

Inserire 1 record con:
- Type: missing.Type (fx/amb)
- Title: candidate.Title
- Source: "freesound"|"pixabay"|"mixkit"
- SourceUrl: candidate.SourceUrl
- DownloadUrl: candidate.DownloadUrl
- LocalPath: path salvato
- License: candidate.License
- Author: candidate.Author
- Tags: merge di:
    - missing.Tags (csv)
    - candidate.Tags (se presenti)
  Normalizzare e deduplicare
- DurationSeconds: candidate.DurationSeconds (se presente)
- ExternalId: candidate.ExternalId
- Notes: "from missingId=<id>; query=<queryUsed>"
- CreatedUtc: now

Aggiornare record mancante:
- Status = "found"
- (opzionale) FoundSoundId = nuovo SoundId
- (opzionale) FoundSource = source
- (opzionale) FoundUrl = candidate.SourceUrl
- (opzionale) Message = "OK"

Se non trovato:
- Status = "failed"
- Message = "No candidates from sources: freesound,pixabay,mixkit"

Se skipped:
- Status = "skipped"
- Message = "type=music"
